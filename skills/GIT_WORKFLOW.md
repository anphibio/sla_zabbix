# GIT_WORKFLOW.md

## Branches

- main: produção
- develop: homologação/desenvolvimento integrado
- feature/nome-da-feature
- bugfix/nome-do-bug
- hotfix/nome-do-hotfix
- release/x.y.z

## Commits

Tipos:

- feat
- fix
- docs
- refactor
- test
- chore
- ci
- security

Exemplo:

```text
feat: adiciona dashboard de auditoria
```

## Pull Request / Merge Request

Toda alteração relevante deve informar:

- Objetivo
- Arquivos principais
- Como testar
- Riscos
- Prints quando alterar UI
